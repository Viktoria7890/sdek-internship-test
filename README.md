# CdekStart — RAG-чатбот программы международной стажировки СДЭК

RAG-чатбот на базе LangGraph и FAISS для ответов на вопросы о программе международной стажировки CdekStart от компании СДЭК. Чатбот умеет уточнять страну стажировки при неоднозначных запросах и отвечает строго на основе базы знаний.

## Почему именно такая реализация

### LangGraph вместо простой цепочки LangChain

Большинство RAG-решений строятся как линейная цепочка: _получить вопрос → найти документы → сгенерировать ответ_. Этого достаточно для простых Q&A, но не для диалогового агента с логикой ветвления.

LangGraph позволяет описать агента как **граф состояний** с явными переходами между нодами. В нашем случае это критично: после поиска документов нужно принять решение — задать уточняющий вопрос или сразу ответить. Без графа это решение пришлось бы зашивать в промпт и надеяться, что LLM его правильно выполнит. С LangGraph логика уточнений — детерминированный код, а не поведение модели.

**Преимущества:**
- Логика ветвления вынесена из промпта в код — предсказуемо и тестируемо
- Граф легко расширять: добавить новую ноду (например, валидацию ответа) — это одна функция и одно ребро
- Состояние агента явное и типизированное (`AgentState`)

### FAISS вместо облачной векторной БД

Для базы знаний из 5 небольших файлов разворачивать Pinecone, Weaviate или Qdrant — избыточно. FAISS работает полностью в памяти, без внешних зависимостей, и индекс строится за доли секунды.

**Преимущества:**
- Нет отдельного сервиса — меньше точек отказа
- Индекс строится при старте приложения прямо из файлов `data/`
- При необходимости масштабирования легко заменить на любую LangChain-совместимую векторную БД

### Ollama как локальная LLM по умолчанию

Проект должен запускаться без внешних API-ключей командой `docker-compose up --build`. Ollama решает эту задачу: модели скачиваются автоматически, всё работает локально.

**Преимущества:**
- Нулевой порог входа — не нужен API-ключ для первого запуска
- Данные не покидают локальную машину
- Легко переключиться на OpenAI через одну переменную в `.env` для лучшего качества

### Контекстно-обогащённый поиск

При формировании запроса к FAISS используется не только текущее сообщение, но и история диалога. Это решает проблему коротких уточняющих сообщений: когда пользователь пишет просто _"Германия"_ — FAISS понимает контекст и находит нужные документы.

## Архитектура

- **FastAPI** — HTTP-сервер с эндпоинтом `POST /chat`
- **LangGraph** — граф агента с логикой уточняющих вопросов
- **FAISS** — векторное хранилище для семантического поиска по документам
- **LangChain** — оркестрация RAG-пайплайна
- **OpenAI / Ollama** — подключаемые LLM-провайдеры
- **Docker Compose** — контейнеризация для деплоя

## Быстрый старт

### Требования

- [Docker Desktop](https://www.docker.com/products/docker-desktop/) (macOS / Windows / Linux)
- Git

### macOS

```bash
# 1. Клонировать репозиторий
git clone <ссылка-на-репозиторий>
cd sdek-internship-test

# 2. Создать файл конфигурации
cp .env.example .env

# 3. Запустить
docker-compose up --build
```

> **Первый запуск** занимает 5–15 минут — Docker скачивает модели Ollama (~2.3 GB).

Сервис будет доступен по адресу: `http://localhost:8000`

```powershell
# 1. Клонировать репозиторий
git clone <ссылка-на-репозиторий>

# 2. Создать файл конфигурации

# 3. Запустить
docker-compose up --build
```

> На Windows убедитесь, что Docker Desktop запущен и включена поддержка WSL 2.

Сервис будет доступен по адресу: `http://localhost:8000`

## Конфигурация LLM

Все настройки задаются через переменные окружения в файле `.env`. Используйте `.env.example` как шаблон.

### Ollama — локальная LLM (по умолчанию, без API-ключей)

```env
LLM_PROVIDER=ollama
OLLAMA_BASE_URL=http://ollama:11434
OLLAMA_MODEL=llama3.2

EMBEDDING_PROVIDER=ollama
OLLAMA_EMBEDDING_MODEL=nomic-embed-text
```

Модели скачиваются автоматически при первом запуске через `docker-compose`.

> **Ограничение:** локальные модели работают на CPU и дают ответы медленнее (~10–30 сек),
> качество текста ниже, чем у GPT. Для production рекомендуется OpenAI.

### OpenAI (лучшее качество ответов)

```env
LLM_PROVIDER=openai
OPENAI_API_KEY=sk-ваш-ключ
OPENAI_MODEL=gpt-4o-mini

EMBEDDING_PROVIDER=openai
OPENAI_EMBEDDING_MODEL=text-embedding-3-small
```

Ключ можно получить на [platform.openai.com](https://platform.openai.com).

После изменения `.env` перезапустите сервис:
```bash
docker-compose down && docker-compose up
```

## Особенности RAG-агента

### 1. Уточняющие вопросы при неоднозначных запросах

Бот автоматически обнаруживает, когда вопрос касается страно-специфичной темы (стипендия, налог, виза, рабочий день), но страна не указана. В этом случае он **задаёт уточняющий вопрос** вместо того чтобы угадывать:

```
Пользователь: "Какая ставка стипендии?"
Бот: "Уточните, пожалуйста, для какой страны: Германия (Берлин) или Франция (Париж)?"
Пользователь: "Германия"
Бот: "Ставка стипендии в Германии — 1200 евро в месяц."
```

### 2. Память контекста в рамках сессии

Бот запоминает упомянутую страну на протяжении всего диалога. Повторно уточнять не нужно:

```
Пользователь: "Германия"
Бот: "Ставка стипендии — 1200 евро."
Пользователь: "А какой налог?"        ← страна не указана
Бот: "Налог в Германии — 15%."        ← запомнил из контекста
```

### 3. Строгий ответ по базе знаний (без галлюцинаций)

Бот отвечает **только на основе документов** из папки `data/`. Если информации нет — честно сообщает об этом:

```
Пользователь: "Какой курс евро к рублю?"
Бот: "К сожалению, у меня нет информации по этому вопросу в базе знаний."
```

### 4. Контекстно-обогащённый поиск

При поиске по векторной базе бот использует не только текущий вопрос, но и историю диалога — это повышает точность поиска релевантных документов при коротких или уточняющих сообщениях.

### LangGraph-граф агента

```
retrieve → check_ambiguity ──▶ clarify → END
                            └─▶ answer  → END
```

- **retrieve** — семантический поиск по FAISS (top-5 документов)
- **check_ambiguity** — проверяет наличие страны в контексте и тему вопроса
- **clarify** — возвращает уточняющий вопрос
- **answer** — формирует ответ через LLM на основе найденных документов

## Тестирование

### Способ 1 — Интерактивный терминал (удобно)

```bash
python3 chat.py
```

Запустится диалоговый интерфейс прямо в терминале (пока `docker-compose` работает в другом окне).

> На Windows: `python chat.py`

### Способ 2 — curl (ручное тестирование)

**Тест 1: уточнение при неоднозначном запросе**
```bash
# Шаг 1 — запрос без страны → должен спросить уточнение
curl -s -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "t1", "message": "какая ставка стипендии?"}' | python3 -m json.tool --no-ensure-ascii

# Шаг 2 — уточняем страну → должен ответить конкретно
curl -s -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "t1", "message": "Германия"}' | python3 -m json.tool --no-ensure-ascii

# Шаг 3 — следующий вопрос без страны → должен помнить контекст
curl -s -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "t1", "message": "а какой налог?"}' | python3 -m json.tool --no-ensure-ascii
```

**Тест 2: страна указана сразу — уточнение не нужно**
```bash
curl -s -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "t2", "message": "какой налог во Франции?"}' | python3 -m json.tool --no-ensure-ascii
```

**Тест 3: общий вопрос без привязки к стране**
```bash
curl -s -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "t3", "message": "когда дедлайн подачи документов?"}' | python3 -m json.tool --no-ensure-ascii
```

**Тест 4: вопрос вне базы знаний — не должен галлюцинировать**
```bash
curl -s -X POST http://localhost:8000/chat \
  -H "Content-Type: application/json" \
  -d '{"session_id": "t4", "message": "какой курс евро?"}' | python3 -m json.tool --no-ensure-ascii
```

Откройте в браузере: **http://localhost:8000/docs**

## База знаний

Документы находятся в директории `data/`:

| Файл | Содержание |
|------|-----------|
| `general_info.txt` | Общая информация о программе CdekStart |
| `deadlines.txt` | Сроки приёма заявок и этапов отбора |
| `benefits.txt` | Льготы и условия для стажёров |
| `germany_rules.txt` | Условия стажировки в Германии (Берлин) |
| `france_rules.txt` | Условия стажировки во Франции (Париж) |

Чтобы добавить новые документы — поместите `.txt`-файл в `data/` и перезапустите сервис.

---

## Ограничения

- **Сессии в памяти** — история диалогов сбрасывается при перезапуске контейнера.
- **Локальные модели медленнее** — ответы на CPU занимают 10–30 секунд, качество ниже чем у GPT.
- **Только `.txt`-файлы** — для поддержки PDF/DOCX нужна доработка `app/rag.py`.
- **FAISS пересоздаётся при каждом запуске** — при большом объёме документов это замедляет старт.
